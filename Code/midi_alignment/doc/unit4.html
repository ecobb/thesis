fmplib.unit4.bin_freqs(fs, fft_len)¶

Returns the center frequency of each bin in the FFT.

Parameters

  • fs – the sample rate

  • fft_len – length of the FFT whose bins are calculated for

Returns

a numpy array of floats listing the center frequency for each of the bins in the first half of the FFT

Parameters

  • fs – the sample rate

  • fft_len – length of the FFT whose bins are calculated for

Returns

fmplib.unit4.cens(chroma, filt_len, ds_rate)¶

Chroma Energy Normalized Statistics - returns a normalized chromagram. Start with a vanilla, un-normalized chromagram. 1. Apply Manhattan norm 2. Quantize 3. Hann window smoothing 4. Downsample 5. Take Euclidean norm

Parameters

  • chroma – the original chromagram

  • filt_len – filter length for time-smoothing

  • ds_rate – downsample factor

Returns

the CENS normalized chromagram

fmplib.unit4.dtw(C, diag_weight=1.0)¶
Dynamic Time Warping - finds the accumlated cost matrix and optimal path from 0,0 to the opposite corner of the cost matrix C.

Uses step sizes (1,0), (1,0), and (1,1).

Parameters

  • C – the cost matrix

  • diag_weight – an optional weight multiplier factor on the diagonal step (1,1).

Returns

(D, path), where D is the accumulated cost matrix and path is an array listing the steps of the optimal-path

fmplib.unit4.make_chromagram(x, fs, fft_len, hop_size, normalize=True, gamma=0, tuning=0.0)¶

Returns the chromagram of signal x.

Parameters

  • x – the signal

  • fs – the sample rate

  • fft_len – the length of the FFT

  • hop_size – the hop_size for the STFT

  • normalize – L^2 normalization if True

  • gamma – optional log compressions (0 = no compression)

  • tuning – an optional MIDI pitch adjustment parameter for alternate tunings, in units of semi-tones

Returns

a matrix of shape (12, num_hops), where num_hops corresponds to the length of the STFT

  • end_pitch – one more than the last pitch value

fmplib.unit4.spec_to_pitch_mtx(fs, fft_len, type='box', tuning=0.0)¶

Create a conversion matrix from a FT vector to a MIDI pitch vector. Is also the conversion matrix from a spectrogram to a pitch-o-gram. AKA, log-frequency spectrum.

Parameters

  • fs – the sample rate

  • fft_len – the length of the FFT

  • type – “box” = straight box window around each frequency range; “tri” = triangular window around each frequency range; “hann” = hanning window around each frequency range

  • tuning – an optional MIDI pitch adjustment parameter for alternate tunings, in units of semi-tones

Returns

a matrix of shape (128, num_bins), where num_bins corresponds to the number of bins in the first half of the FFT
